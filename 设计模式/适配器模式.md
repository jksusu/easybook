## 适配器模式

> database  数据库连接目录<br/>
  │  ├─mysql     链接方式<br/>
  │  ├─mysqli    链接方式<br/>
  │  ├─pdo       链接方式<br/>
  │  │<br/>
  ├─Database.php          链接接口
  
  以上是我们的目录结构，链接 mysql 的三种方法，用一个适配器模式封装一下。
  
  ```php
  <?php
  interface Database
  {
      //定义一个接口，三个方法需要实现。
      function connect($host,$user,$password,$databaseName);
      function query($sql);
      function close();
  }
  ```
  ### 实现一个 mysql 链接方式
  
```php
<?php
class mysql implements Database
{
    //实现这个接口
    function connect($host,$user,$password,$databaseName){}
    function query($sql){}
    function close(){}
}
  
```

### 结论
> 适配器模式能将截然不同的函数封装成统一的api，并对外提供服务。
以上代码封装了操作数据库的统一方法，这样在外部调用，不管我们使用的是什么链接函数，都有统一的对外api。
适配器模式还适用于缓存，redis,file,memcache ..... 可以把这几个封装成一个统一对外的api,这样你的
程序就可以适配多种其他的应用。

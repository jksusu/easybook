  ## 单例模式
  * 单例类只能有一个实例
  * 单例类必须自己创建自己的实例
  * 单例类必须给所有对象提供这一实例
  
  通常在一个项目中一个功能点会用但各种各样的类。这样就会反复创建对象，造成资源的浪费，并且项目结构层次不明了，所以出现了单例模式。
  
  ```php
  <?php 
  //不使用单例模式实例化一个链接数据库的对象
  $getConnection = new Database();
  //以上代码如果系统中有 100个地方要用到，那么他会创建 100个对象。
  
  //正确姿势 使用单例模式例子
  
  // 1. 创建一个数据库连接类
  class Database
  {
       protected static $db;
  	 
  	 //这里一定要是私有的，只能在类的内部访问，这样避免在其他地方 new 对象（符合了以上第二条规定）
  	 private function __construct()
  	 {
  	     //这里是连接数据库的操作，这里不写了。。。。。。。。。。。
  	 }
  	 
  	 // 获取数据库连接单例
  	 //这里 static 修饰为了无需实例化即可访问，通过 ::
       static function getInstance()
  	 {
  	     //判断是否有链接，如果有就返回，没有则创建链接对象
  	     if (self::$db){
                return self::$db;
  		 } else {
  		      // 实例化自己 
  		      self::$db = new self();
  			  return self::$db;
  		 }
  	 }
  }
  
  //这样调用就是单例模式，始终只有一个数据库连接
  $getConnection = Database::getInstance();
  
  
  ```

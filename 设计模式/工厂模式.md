##工厂模式
顾名思义，工厂模式就是一个工厂。工厂是用来生产东西的。
要生产汽车，你的去生产汽车的工厂，生产手机你的去手机工厂。<br/>
放到编程中就是，<br/>
如果你程序需要链接数据库，正常思想就是在哪里用在哪里链接。直接在需要用的地方new对象<br/>
但是如果你的程序有100个地方需要链接数据库操作了，你总不可能在写100编链接数据库的方法吧！
##正确打开方式
创建一个工厂类，其中创建一个 function 就叫 createDatabase,把创建数据库链接的代码写在 createDatabase中，
在需要链接数据库的地方直接调用这个 function 就行了。<br/>
这就是工厂模式，把一个复用多，且共用性高的东西放到一个工厂中去生产。而你不用管他具体是怎么链接上数据库的。

---
##优点
*调用者想要创建对象只需要知道其名称<br/>
*扩展性高，增加功能只需要新增一个工厂即可<br/>
*调用者不用关心其具体的实现。<br/>
##缺点
在一个庞大系统中，肯定是有上百个工厂类，增加了系统的复杂度，同时增加了和调用工厂方法类的依赖。
在一个庞大的系统中这不算什么好事！！！！特别是在一个员工流失率高得团队中！！！！
